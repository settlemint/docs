## Overview and use cases

BSC is designed to provide a programmable blockchain infrastructure with smart contract functionality while maintaining high throughput and low costs. Key characteristics include:

- **EVM Compatibility**: Full compatibility with Ethereum Virtual Machine, allowing easy migration of Ethereum dApps
- **Low Transaction Fees**: Significantly lower gas fees compared to Ethereum mainnet (typically less than $0.10 per transaction)
- **Fast Block Times**: 3-second block time enabling quick transaction confirmations
- **Large Ecosystem**: Extensive DeFi, NFT, and gaming ecosystem with thousands of active projects
- **Cross-Chain Compatibility**: Native support for cross-chain transfers with Binance Chain

### Common use cases

- **DeFi Applications**: Decentralized exchanges, lending protocols, yield farming platforms
- **NFT Marketplaces**: Lower minting and trading costs make it ideal for NFT projects
- **GameFi**: Blockchain gaming applications benefit from fast and cheap transactions
- **Payment Solutions**: Suitable for micropayments and everyday transactions
- **Cross-Chain Bridges**: Hub for multi-chain applications and asset transfers

## Mainnet and testnet

SettleMint supports both the BSC **Mainnet** and the **BSC Testnet**.

The Mainnet is the primary public BSC production blockchain, where actual-value transactions take place. Each transaction requires payment of a transaction fee, payable in the native coin BNB. The Testnet is an instance of the blockchain to be used for testing and experimentation. There are also coins used in the Testnet but they have no value, so there is no risk of real funds.

You can consider the Testnet as a prototype and the Mainnet as the official production blockchain. Or think of this as an analog to production versus staging servers.

### Network details

**BSC Mainnet:**
- Chain ID: 56
- Currency: BNB
- Block Explorer: [BscScan](https://bscscan.com)

**BSC Testnet:**
- Chain ID: 97
- Currency: tBNB (Test BNB)
- Block Explorer: [BscScan Testnet](https://testnet.bscscan.com)

## Setting up BSC nodes in SettleMint

### Prerequisites

Before setting up BSC nodes, ensure you have:
- A SettleMint account with appropriate permissions
- Understanding of blockchain node basics
- Sufficient resources allocated for node deployment
- SettleMint CLI installed (`npm install -g @settlemint/cli`)

### Creating BSC nodes via CLI

You can quickly create BSC nodes using the SettleMint CLI:

```bash
# Create a BSC mainnet node
settlemint node create --blockchain bsc --network mainnet --name my-bsc-node

# Create a BSC testnet node
settlemint node create --blockchain bsc --network testnet --name my-bsc-testnet

# Create an archive node with custom resources
settlemint node create \
  --blockchain bsc \
  --network mainnet \
  --name my-bsc-archive \
  --type archive \
  --cpu 16 \
  --memory 32 \
  --storage 4000
```

### Step-by-step setup guide

#### 1. Create a new BSC node

1. Navigate to the **Blockchain Nodes** section in your SettleMint dashboard
2. Click **"Add Node"** or **"Create New Node"**
3. Select **"BSC (Binance Smart Chain)"** from the available blockchain options
4. Choose between **Mainnet** or **Testnet** based on your requirements

#### 2. Configure node settings

Configure your BSC node with the following parameters:

- **Node Name**: Provide a descriptive name for your node
- **Node Type**: Select the appropriate node type:
  - **Full Node**: Stores complete blockchain data
  - **Archive Node**: Stores all historical states (requires more storage)
- **Region**: Choose the deployment region closest to your users
- **Resources**: Select appropriate CPU, memory, and storage based on your needs

#### 3. Network configuration

Configure network settings for your BSC node:

- **RPC Endpoint**: Enable/disable public RPC access
- **WebSocket Endpoint**: Enable for real-time event subscriptions
- **API Rate Limiting**: Configure rate limits to prevent abuse
- **Whitelisted IPs**: Restrict access to specific IP addresses (recommended for production)

#### 4. Deploy and synchronize

1. Review your configuration settings
2. Click **"Deploy Node"** to begin the deployment process
3. Monitor the deployment status in the dashboard
4. Wait for the node to synchronize with the BSC network (this may take several hours for mainnet)

### Configuration options

#### Advanced settings

- **State Pruning**: Enable to reduce storage requirements (not recommended for archive nodes)
- **Snapshot Sync**: Use snapshots for faster initial synchronization
- **Custom Genesis**: For private BSC-based networks
- **Peering Configuration**: Manage peer connections and discovery

#### Performance optimization

- **Cache Size**: Adjust cache settings based on available memory
- **Database Backend**: Choose between LevelDB or PebbleDB
- **Concurrent Downloads**: Set parallel block download limits
- **Transaction Pool Size**: Configure mempool capacity

## Using the SettleMint SDK with BSC

### Installation

Install the SettleMint SDK with viem support:

```bash
npm install @settlemint/sdk-viem viem
```

### Connecting to BSC nodes

```typescript
import { createSettleMintClient } from '@settlemint/sdk-viem';
import { bsc, bscTestnet } from 'viem/chains';

// Connect to BSC Mainnet
const mainnetClient = createSettleMintClient({
  chain: bsc,
  transport: 'https://your-bsc-node-url.settlemint.io',
  apiKey: 'your-api-key'
});

// Connect to BSC Testnet
const testnetClient = createSettleMintClient({
  chain: bscTestnet,
  transport: 'https://your-bsc-testnet-url.settlemint.io',
  apiKey: 'your-api-key'
});
```

### Common BSC operations

#### Reading blockchain data

```typescript
// Get current block number
const blockNumber = await mainnetClient.getBlockNumber();
console.log('Current block:', blockNumber);

// Get BNB balance
const balance = await mainnetClient.getBalance({
  address: '0x742d35Cc6634C0532925a3b844Bc9e7595f89c0d'
});
console.log('Balance:', balance, 'wei');

// Get transaction details
const transaction = await mainnetClient.getTransaction({
  hash: '0x...'
});

// Watch for new blocks
const unwatch = mainnetClient.watchBlockNumber({
  onBlockNumber: (blockNumber) => {
    console.log('New block:', blockNumber);
  }
});
```

#### Interacting with smart contracts

```typescript
// Read from a contract
const result = await mainnetClient.readContract({
  address: '0x...',
  abi: contractABI,
  functionName: 'balanceOf',
  args: ['0x742d35Cc6634C0532925a3b844Bc9e7595f89c0d']
});

// Write to a contract (requires wallet)
import { privateKeyToAccount } from 'viem/accounts';

const account = privateKeyToAccount('0x...');

const hash = await mainnetClient.writeContract({
  address: '0x...',
  abi: contractABI,
  functionName: 'transfer',
  args: ['0xrecipient...', BigInt(1000000000000000000)], // 1 BNB
  account
});

// Wait for confirmation
const receipt = await mainnetClient.waitForTransactionReceipt({ hash });
```

#### Event monitoring

```typescript
// Watch contract events
const unwatch = mainnetClient.watchContractEvent({
  address: '0x...',
  abi: contractABI,
  eventName: 'Transfer',
  onLogs: (logs) => {
    logs.forEach(log => {
      console.log('Transfer event:', log.args);
    });
  }
});

// Get historical events
const events = await mainnetClient.getContractEvents({
  address: '0x...',
  abi: contractABI,
  eventName: 'Transfer',
  fromBlock: BigInt(25000000),
  toBlock: 'latest'
});
```

#### Working with BEP-20 tokens

```typescript
// BEP-20 is the same as ERC-20 on BSC
const tokenABI = [
  'function balanceOf(address) view returns (uint256)',
  'function transfer(address to, uint256 amount) returns (bool)',
  'function approve(address spender, uint256 amount) returns (bool)',
  'function allowance(address owner, address spender) view returns (uint256)'
];

// Get token balance
const tokenBalance = await mainnetClient.readContract({
  address: '0xe9e7CEA3DedcA5984780Bafc599bD69ADd087D56', // BUSD
  abi: tokenABI,
  functionName: 'balanceOf',
  args: [userAddress]
});

// Transfer tokens
const transferHash = await mainnetClient.writeContract({
  address: '0xe9e7CEA3DedcA5984780Bafc599bD69ADd087D56',
  abi: tokenABI,
  functionName: 'transfer',
  args: [recipientAddress, amount],
  account
});
```

### Gas estimation and optimization

```typescript
// Estimate gas for a transaction
const gasEstimate = await mainnetClient.estimateGas({
  to: '0x...',
  value: BigInt(1000000000000000000), // 1 BNB
  data: '0x...'
});

// Get current gas price
const gasPrice = await mainnetClient.getGasPrice();
console.log('Current gas price:', gasPrice, 'wei');

// Send transaction with custom gas settings
const txHash = await mainnetClient.sendTransaction({
  to: '0x...',
  value: BigInt(1000000000000000000),
  gas: gasEstimate,
  gasPrice: gasPrice * BigInt(110) / BigInt(100), // 10% higher
  account
});
```

## Best practices

### Security recommendations

1. **Access Control**: Always use IP whitelisting for production nodes
2. **HTTPS/WSS**: Enable SSL/TLS for all external connections
3. **Private Keys**: Never store private keys on public nodes
4. **Regular Updates**: Keep your nodes updated with the latest BSC client versions
5. **Monitoring**: Set up alerts for node health and performance metrics

### Performance optimization

1. **Resource Allocation**: 
   - Minimum: 4 CPU cores, 8GB RAM, 500GB SSD
   - Recommended: 8 CPU cores, 16GB RAM, 1TB NVMe SSD
   - Archive Node: 16 CPU cores, 32GB RAM, 4TB+ NVMe SSD

2. **Network Optimization**:
   - Use dedicated bandwidth for node synchronization
   - Enable compression for peer connections
   - Configure appropriate peer limits

3. **Storage Management**:
   - Use SSDs for blockchain data
   - Regular cleanup of log files
   - Monitor disk usage and plan for growth

### High availability setup

For production environments, consider:

1. **Multiple Nodes**: Deploy multiple nodes across different regions
2. **Load Balancing**: Distribute requests across nodes
3. **Failover Configuration**: Automatic switching to backup nodes
4. **Regular Backups**: Scheduled snapshots of node data

## Troubleshooting

### Common issues and solutions

#### Node not syncing

**Problem**: Node stuck at a specific block height

**Solutions**:
1. Check network connectivity and firewall rules
2. Verify sufficient disk space
3. Restart the node service
4. Check for chain reorganizations
5. Use `--syncmode snap` for faster synchronization

#### High resource usage

**Problem**: Excessive CPU or memory consumption

**Solutions**:
1. Adjust cache sizes in configuration
2. Enable state pruning if running a full node
3. Limit peer connections
4. Check for memory leaks in logs

#### RPC connection issues

**Problem**: Cannot connect to RPC endpoint

**Solutions**:
1. Verify RPC is enabled in node configuration
2. Check firewall rules and security groups
3. Confirm correct endpoint URL and port
4. Test connection with the SDK:

```typescript
import { createSettleMintClient } from '@settlemint/sdk-viem';
import { bsc } from 'viem/chains';

async function testConnection() {
  try {
    const client = createSettleMintClient({
      chain: bsc,
      transport: 'https://your-node-url.settlemint.io',
      apiKey: 'your-api-key'
    });
    
    const blockNumber = await client.getBlockNumber();
    console.log('✅ Connection successful! Block:', blockNumber);
  } catch (error) {
    console.error('❌ Connection failed:', error.message);
    
    // Common issues:
    // - Invalid API key
    // - Incorrect URL
    // - Network restrictions
    // - Node not fully synced
  }
}
```

#### Peers not connecting

**Problem**: Low peer count or no peers

**Solutions**:
1. Check if node is accessible from the internet
2. Verify bootnode configuration
3. Ensure ports 30303 (TCP/UDP) are open
4. Add static peers if needed

### Debug commands

Useful SDK methods for troubleshooting:

```typescript
import { createSettleMintClient } from '@settlemint/sdk-viem';
import { bsc } from 'viem/chains';

const client = createSettleMintClient({
  chain: bsc,
  transport: 'https://your-bsc-node-url.settlemint.io',
  apiKey: 'your-api-key'
});

// Check sync status
const syncStatus = await client.request({
  method: 'eth_syncing'
});
console.log('Sync status:', syncStatus);

// Get peer count
const peerCount = await client.request({
  method: 'net_peerCount'
});
console.log('Connected peers:', parseInt(peerCount, 16));

// Check latest block
const latestBlock = await client.getBlock({
  blockTag: 'latest'
});
console.log('Latest block:', latestBlock.number);

// Get node info
const nodeInfo = await client.request({
  method: 'web3_clientVersion'
});
console.log('Node version:', nodeInfo);

// Check chain ID
const chainId = await client.getChainId();
console.log('Chain ID:', chainId); // Should be 56 for mainnet, 97 for testnet

// Monitor node health
async function checkNodeHealth() {
  try {
    const [blockNumber, peerCount, syncStatus] = await Promise.all([
      client.getBlockNumber(),
      client.request({ method: 'net_peerCount' }),
      client.request({ method: 'eth_syncing' })
    ]);
    
    return {
      blockNumber,
      peers: parseInt(peerCount, 16),
      syncing: syncStatus !== false,
      healthy: true
    };
  } catch (error) {
    return { healthy: false, error: error.message };
  }
}
```

## Consensus mechanism

BSC uses a **Proof of Staked Authority (PoSA)** consensus mechanism, which combines elements of Proof of Authority (PoA) and Delegated Proof of Stake (DPoS). This hybrid approach provides:

- **21 Validators**: A fixed set of validators elected by BNB stakers
- **Block Time**: 3-second block time for fast confirmations
- **Epoch System**: Validators rotate every 24 hours (epoch)
- **Slashing**: Malicious validators can be slashed and lose their staked BNB
- **Rewards**: Block rewards and transaction fees distributed to validators and delegators

The PoSA consensus ensures high throughput while maintaining a reasonable level of decentralization through the staking mechanism.

More information on BSC's consensus can be found on the [official BNB Chain documentation](https://docs.bnbchain.org/docs/learn/consensus).
