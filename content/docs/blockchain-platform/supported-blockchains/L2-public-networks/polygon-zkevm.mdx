### How zkEVM Processes Transactions

<Steps>
  <Step>
    **Transaction Submission**
    
    Users submit transactions to the zkEVM sequencer, which batches them for efficient processing.
  </Step>
  
  <Step>
    **Execution & Proof Generation**
    
    The zkEVM executes transactions and generates zero-knowledge proofs proving correct computation.
  </Step>
  
  <Step>
    **Batch Submission**
    
    Transaction batches and their validity proofs are submitted to Ethereum mainnet for final settlement.
  </Step>
  
  <Step>
    **Cryptographic Verification**
    
    Ethereum verifies the zero-knowledge proof, confirming all transactions in the batch are valid.
  </Step>
  
  <Step>
    **Instant Finality**
    
    Once verified on Ethereum, transactions achieve irreversible finality with mathematical certainty.
  </Step>
</Steps>

## How does Proof of Efficiency secure the zkEVM network?

Polygon zkEVM introduces Proof of Efficiency (PoE), a novel consensus mechanism that incentivizes participants to generate zero-knowledge proofs efficiently while maintaining decentralization and security.

### Proof of Efficiency Architecture

  <Card title="Competitive Proving" description="Validators compete to generate zero-knowledge proofs, optimizing for both speed and cost efficiency." />

### Network Participant Roles

<Accordions>
  <Accordion title="Sequencers: Transaction Ordering">
    **Primary Function**: Collect user transactions and create batches for processing
    
    **Economic Model**: Pay gas fees to Ethereum for batch submission, earn transaction fees
    
    **Competition**: Multiple sequencers ensure decentralization and prevent censorship
    
    **Requirements**: MATIC stake to participate, computational resources for batching
  </Accordion>
  
  <Accordion title="Aggregators: Proof Generation">
    **Primary Function**: Generate zero-knowledge proofs for transaction batches
    
    **Economic Model**: Earn MATIC rewards for valid proof generation
    
    **Competition**: Fastest, most efficient proof generators earn more rewards
    
    **Requirements**: Specialized hardware for ZK proof computation, MATIC stake
  </Accordion>
  
  <Accordion title="Validators: Network Security">
    **Primary Function**: Verify proofs and maintain network consensus
    
    **Economic Model**: Earn staking rewards and transaction fees
    
    **Responsibilities**: Monitor network health, dispute invalid proofs
    
    **Requirements**: MATIC stake, reliable infrastructure, cryptographic verification
  </Accordion>
</Accordions>
